# Day 1 실습 — 내 프로덕트 랜딩 페이지 만들기

## 사전 준비

아래가 설치되어 있어야 합니다.
- Claude Code (혹은 Codex)
- Node.js
- Git

## Step 1. Superpowers 설치

Claude Code를 열고 아래를 입력합니다:

```
/plugin install superpowers
```

설치가 끝나면 새 채팅을 시작합니다.

## Step 2. 프로젝트 폴더 만들기

```bash
mkdir product-woongjae
cd product-woongjae
git init
```

## Step 3. 프로덕트 정의하기

Claude Code에게 아래와 같이 말합니다:

> "내가 만들 프로덕트의 랜딩 페이지를 만들고 싶어.
> 먼저 어떤 프로덕트인지 같이 정리해줘."

Superpowers(또는 다른 Harness)가 활성화되어 있으면, 바로 만들지 않고 질문부터 던집니다:
- 누구를 위한 프로덕트인가?
- 어떤 문제를 푸는가?
- 핵심 기능은 무엇인가?

이 질문들에 답하면서 프로덕트의 윤곽을 잡습니다.

아직 프로덕트 아이디어가 없다면, 아래 질문에 먼저 답해보세요:

```
최근 1주일간 가장 반복적으로 답답했던 문제는?
그 문제를 해결하는 가장 작은 도구가 있다면?
```

## Step 4. 랜딩 페이지 만들기

프로덕트 정의가 끝나면:

> "이 프로덕트의 한 장짜리 랜딩 페이지를 만들어줘."

Superpowers (또는 다른 Harness)가 계획을 세우고, 구조를 잡고, 만들어줍니다.

## Step 5. 확인하기

로컬에서 확인합니다:

```bash
npx serve .
```

브라우저에서 열어서 내 랜딩 페이지가 보이는지 확인합니다.

## Step 6. (선택) Vercel 배포

시간이 되면 배포까지 해봅니다:

```bash
npx vercel
```

배포가 끝나면 URL이 나옵니다. 이 URL을 다른 사람에게 보내볼 수 있습니다.

## 셀프 체크

실습이 끝나면 아래를 확인해보세요:

- [ ] 내 프로덕트가 누구의 어떤 문제를 푸는지 한 줄로 말할 수 있는가?
- [ ] 랜딩 페이지에 핵심 동작 1개가 명확히 드러나는가?
- [ ] Harness가 내가 인지하지 못 했던 영역에 대해 드러내어 주었는가?
- [ ] 만약 그러지 않았다면 무엇이 필요하다고 생각하는가?
